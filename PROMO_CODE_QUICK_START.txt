## ⏳ What's LEFT (3 Steps to Production)

### Step 1️⃣: Create Firestore Config (5 min)

**Via Firebase Console:**
1. Go to Firestore
2. Create collection: **`config`**
3. Add document: **`promo_codes`**
4. Copy-paste JSON:

```json
{
  "codes": {
    "MAS10": {
      "type": "percentage",
      "value": 10,
      "minSubtotalCents": 5000,
      "disabled": false
    },
    "MEDIA5": {
      "type": "fixed",
      "value": 500,
      "minSubtotalCents": 2000,
      "disabled": false
    },
    "WELCOME20": {
      "type": "percentage",
      "value": 20,
      "maxDiscountCents": 2000,
      "disabled": false
    }
  }
}
```

## 🔗 Integration Points

**Client → Server:**
```
_applyPromo()
  ↓
CartCheckoutService.validatePromoCode()
  ↓
Firebase Function: validatePromoCode()
  ↓
Firestore: config/promo_codes (read)
```

**Checkout → Server:**
```
_checkout()
  ↓
startMixedCheckout(promoCode: _promoCode)
  ↓
Firebase Function: createMixedCartPaymentIntent()
  ↓
Firestore: orders/{storeOrderId} (write with promo info)
  ↓
Stripe: PaymentIntent(amount=DISCOUNTED_TOTAL)
```

---

## 🚨 Important Notes

1. **Firestore config is REQUIRED** - CF won't find codes without it
2. **Server-side validation is paramount** - Never trust client amount
3. **Metadata in Stripe PaymentIntent** - Allows audit trail
4. **User must be authenticated** - CF checks `req.auth`
5. **No rate limiting yet** - Can add if promo abuse detected
